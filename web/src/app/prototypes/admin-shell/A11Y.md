# A11Y.md — Tulala Accessibility Guide

> **WS-17.6** · Keyboard map, ARIA conventions, and screen-reader patterns for the Tulala admin prototype.
> Target: WCAG 2.1 Level AA.

---

## 1. Keyboard navigation map

| Key | Context | Action |
|---|---|---|
| `Tab` / `Shift+Tab` | Global | Focus next / previous interactive element |
| `Enter` / `Space`   | Button | Activate |
| `Escape`            | Drawer / modal / popover | Close |
| `↑` / `↓`           | List rows (inbox, roster) | Navigate between rows |
| `j` / `k`           | Inbox (not in modal) | Next / previous row |
| `e`                 | Inbox row focused | Archive |
| `r`                 | Inbox row focused | Reply |
| `c`                 | Global (not in modal) | Compose new inquiry |
| `g + i`             | Global | Go to inbox/messages |
| `g + o`             | Global | Go to overview |
| `g + c`             | Global | Go to calendar |
| `g + t`             | Global | Go to talent/roster |
| `?`                 | Global | Open shortcut help overlay |
| `Cmd+K` / `Ctrl+K`  | Global | Open command palette |

## 2. ARIA conventions

### Roles
- `role="dialog" aria-modal="true"` on every drawer / modal panel
- `role="status"` with `aria-live="polite"` on `StaleDataPill` (non-urgent)
- `role="alert"` with `aria-live="assertive"` on error toasts
- `role="navigation"` on the main sidebar and bottom nav
- `role="main"` on the primary surface area (done in `SurfaceRouter`)

### Labels
- Every icon-only button MUST have `aria-label="..."` (no exceptions)
- Every form input MUST have a visible `<label>` or `aria-label`
- `aria-pressed` on toggle buttons to reflect current state
- `aria-expanded` on collapsible sections
- `aria-invalid="true"` on fields with validation errors (via `FieldError`)

### Live regions
- Toast stack: `role="status"` / `aria-live="polite"` for default/info/success
- Error toasts: `aria-live="assertive"`
- Stale-data pill: `role="status"` `aria-live="polite"`
- New message counts: update via `aria-live="polite"` region in topbar

## 3. Focus management

### Drawers
- On open: move focus to the drawer's first focusable element
- On close: return focus to the element that triggered the open
- Trap focus within the drawer while open (Tab cycles inside)

### Modals / dialogs
- Same rules as drawers
- Backdrop click / Esc close must also restore focus

### Page navigation
- On page change: move focus to `<h1>` of the new page (or the `PageHeader`)
- Keyboard shortcut navigation (`g+i`, etc.): move focus to the new page's heading

## 4. Color and visual signals

**Rule:** every status signal must have BOTH a color AND a text/icon label.
Color alone is insufficient under WCAG 1.4.1.

| ❌ Color only | ✅ Color + label |
|---|---|
| Green dot | Green dot + "Active" |
| Red border | Red border + error message text |
| Orange progress bar | Orange bar + "80% used" |

Chips, badges, and `CapsLabel` all include text — they pass this rule automatically.

## 5. Reduced motion

Use `useReducedMotion()` from `_primitives.tsx` before any animation:

```tsx
const reduced = useReducedMotion();
style={{ transition: reduced ? "none" : "opacity .2s ease" }}
```

Affected components:
- Drawer open/close slide
- Toast enter/exit
- Page transition fade
- StaleDataPill entrance animation
- GuidedTour spotlight entrance

---

## 6. Skip links

Every surface should have a skip-to-main-content link as the first focusable element:

```html
<a href="#tulala-main" class="skip-link">Skip to main content</a>
```

The `id="tulala-main"` is set in `SurfaceRouter`.

## 7. Screen-reader tested patterns

### Inquiry row
```html
<button role="row" aria-label="Inquiry from Versace — Offer pending — 2 unread — needs your decision">
```

### Drawer
```html
<aside role="dialog" aria-modal="true" aria-label="New inquiry">
  <!-- focus trap active -->
</aside>
```

### Toast
```html
<div role="status" aria-live="polite">
  Counter sent to coordinator
</div>
```
